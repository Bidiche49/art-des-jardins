# BUG-022: Images blog-* cassees sur 6 articles (chemins en dur)

**Type:** Bug
**Statut:** Fait
**Priorite:** Haute
**Complexite:** S
**Tags:** ui, vitrine
**Date creation:** 2026-02-28

## Description

6 articles de blog utilisent des images stockees dans `/images/blog/` (images Unsplash dediees), mais les pages blog construisent les chemins d'images **en dur** avec `/images/realisations/`. Les images ne s'affichent donc pas.

## Cause racine

Les fichiers suivants construisent les chemins d'images en dur au lieu d'utiliser `getImage()` du manifest :

### `apps/vitrine/src/app/conseils/[slug]/page.tsx`
- **Ligne ~31** (OpenGraph) : `` url: `/images/realisations/${article.imageSlug}-1200w.webp` ``
- **Ligne ~109** (JSON-LD schema) : `` image: `https://art-et-jardin.fr/images/realisations/${article.imageSlug}-1200w.webp` ``
- **Ligne ~186** (Hero section) : `` src={`/images/realisations/${article.imageSlug}-1920w.webp`} ``
- **Ligne ~281** (CTA section) : `` src={`/images/realisations/${article.imageSlug}-1200w.webp`} ``

### `apps/vitrine/src/app/conseils/page.tsx`
- **Ligne ~44** (Cards articles) : `` src={`/images/realisations/${article.imageSlug}-800w.webp`} ``

## Correction

Remplacer les chemins en dur par l'utilisation de `getImage(article.imageSlug)` du fichier `images-manifest.ts`, qui contient les chemins corrects pour chaque image (y compris les blog-*).

Le manifest a deja les bonnes entrees avec les bons chemins (`/images/blog/...`). Il suffit de :
1. Importer `getImage` depuis `@/lib/images-manifest`
2. Remplacer chaque chemin en dur par `getImage(article.imageSlug).sizes[TAILLE].path`
