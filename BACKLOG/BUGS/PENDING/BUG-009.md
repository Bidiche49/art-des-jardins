## Description

Le formulaire de contact utilise Web3Forms comme service d'envoi, mais la cle API est en dur avec la valeur placeholder `'YOUR_ACCESS_KEY'`. Le formulaire ne fonctionne donc PAS en production - les soumissions echouent silencieusement ou renvoient une erreur.

C'est **bloquant pour la conversion** : le site genere du trafic SEO mais aucun lead ne peut etre capture.

## Fichiers concernes

- `apps/vitrine/src/components/ContactForm.tsx` (ligne 76)

## Code problematique

```typescript
access_key: process.env.NEXT_PUBLIC_WEB3FORMS_KEY || 'YOUR_ACCESS_KEY',
```

## Approche

1. Creer un compte Web3Forms (https://web3forms.com) - gratuit jusqu'a 250 soumissions/mois
2. Obtenir l'access key
3. Configurer la variable d'environnement `NEXT_PUBLIC_WEB3FORMS_KEY` dans :
   - `.env.local` pour le dev
   - Variables d'environnement du serveur de production
4. Configurer l'email de destination dans Web3Forms (artdesjardins49@gmail.com)
5. Tester l'envoi en dev et en production
