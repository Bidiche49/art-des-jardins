# IMP-086: Reduire opacite hero overlays pour mieux voir les photos

**Type:** Improvement
**Statut:** A faire
**Priorite:** Haute
**Complexite:** XS
**Tags:** vitrine, design, ux
**Date creation:** 2026-02-27

---

## Description

Les hero overlays sont trop sombres. Le overlay par defaut utilise `rgba(35,50,25, 0.8)` soit 80% d'opacite, ce qui rend les photos de realisations (argument de vente n.1 pour un paysagiste) a peine visibles.

Pour un site premium paysagiste, les photos doivent etre le hero, pas le texte. Il faut trouver un meilleur equilibre entre lisibilite du texte et visibilite de l'image.

### Valeurs actuelles (globals.css)
- `.hero-overlay` : 0.8 → 0.65 → 0.2 → 0.05
- `.hero-overlay-strong` : 0.88 → 0.7 → 0.25 → 0.08
- `.hero-overlay-light` : 0.7 → 0.45 → 0.1 → 0

### Valeurs proposees
- `.hero-overlay` : 0.55 → 0.45 → 0.15 → 0.05
- `.hero-overlay-strong` : 0.7 → 0.55 → 0.2 → 0.05
- `.hero-overlay-light` : 0.5 → 0.3 → 0.08 → 0

Le text-shadow sur les titres hero compense la reduction d'opacite pour maintenir la lisibilite.

## Criteres d'acceptation
- [ ] Photos hero clairement visibles
- [ ] Texte hero toujours lisible (contraste WCAG AA)
- [ ] Test sur toutes les pages hero (homepage, services, contact, villes)
- [ ] Test mobile (ecrans plus petits, contraste important)
