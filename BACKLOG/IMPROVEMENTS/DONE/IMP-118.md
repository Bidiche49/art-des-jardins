# IMP-118: Varier les images de fond des sections CTA

**Type:** Improvement
**Statut:** Fait
**Priorité:** Basse
**Complexité:** XS
**Tags:** ux, ui
**Date création:** 2026-02-28

---

## Description

6 pages/templates utilisent `creation-7-1200w.webp` comme image de fond CTA. Un prospect qui visite 3-4 pages voit le même bandeau — l'oeil s'habitue et ignore.

3 pages ont déjà une image contextuelle (élagage, abattage, entretien Angers).

## Pages à modifier

| Page | Image actuelle | Image proposée |
|------|---------------|----------------|
| `page.tsx` (homepage) | creation-7 | creation-7 (OK, c'est la landing principale) |
| `a-propos/` | creation-7 | image chantier/équipe |
| `paysagiste-angers/` | creation-7 | image aménagement variée |
| `services/[slug]/` | creation-7 | **dynamique** selon le service (chaque service a déjà un imageSlug) |
| `conseils/[slug]/` | creation-7 | **dynamique** selon la catégorie ou l'imageSlug de l'article |
| `[serviceCity]/` | creation-7 | varier (rotation ou image générique différente de homepage) |

## Point d'attention

Pour `services/[slug]/` et `conseils/[slug]/`, privilégier une image dynamique liée au contenu plutôt qu'une image statique différente. Les données existent déjà (imageSlug dans services-data et blog-data).

## Critères d'acceptation
- [ ] Chaque page/template a une image CTA pertinente par rapport à son contenu
- [ ] Les pages services utilisent l'image du service correspondant
- [ ] Les articles blog utilisent une image liée à la catégorie ou à l'article
- [ ] Pas deux pages consécutives dans un parcours type avec la même image CTA
- [ ] Les images existent déjà dans /images/realisations/ (pas de nouvelles images)

## Fichiers concernés
- `apps/vitrine/src/app/a-propos/page.tsx`
- `apps/vitrine/src/app/paysagiste-angers/page.tsx`
- `apps/vitrine/src/app/services/[slug]/page.tsx`
- `apps/vitrine/src/app/conseils/[slug]/page.tsx`
- `apps/vitrine/src/app/[serviceCity]/page.tsx`
