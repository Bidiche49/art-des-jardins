# IMP-083: Ajouter composant Testimonials sur la homepage

**Type:** Improvement
**Statut:** A faire
**Priorite:** Critique
**Complexite:** XS
**Tags:** vitrine, cro, conversion, ux

## Description

Le composant `Testimonials.tsx` existe et contient 6 temoignages clients avec notes, mais il n'est **PAS importe ni affiche sur la homepage**. La page d'accueil — page principale de conversion — n'a aucune preuve sociale client.

C'est un manque critique en CRO : les temoignages sont le signal de confiance n.1 pour les services a domicile. Impact estime : -15 a -25% de conversions.

## User Story
**En tant que** visiteur du site
**Je veux** voir des avis de clients precedents sur la page d'accueil
**Afin de** me rassurer sur la qualite du service avant de demander un devis

## Criteres d'acceptation
- [ ] Composant Testimonials importe et affiche sur la homepage
- [ ] Place entre la galerie (ou Before/After) et la FAQ
- [ ] Schema.org AggregateRating + Review present sur la homepage
- [ ] Responsive mobile (1 colonne) et desktop (3 colonnes)
- [ ] Note aggregate visible (4.8/5)

## Fichiers concernes
- `apps/vitrine/src/app/page.tsx` (ajouter import + placement)
- `apps/vitrine/src/components/Testimonials.tsx` (deja existant)

## Analyse / Approche
Simple import + ajout dans le JSX de page.tsx. Le composant est deja complet avec schema.org integre.
