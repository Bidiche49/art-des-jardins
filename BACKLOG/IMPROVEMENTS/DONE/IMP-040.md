## Description

Le formulaire de contact ne demande pas la ville ou le code postal du visiteur. Cette information est essentielle pour un artisan paysagiste :
- Permet de verifier que le client est dans la zone d'intervention (30 km autour d'Angers)
- Permet de qualifier le lead plus rapidement
- Evite les echanges email inutiles pour determiner la localisation

## User Story

**En tant que** paysagiste
**Je veux** connaitre la localisation du prospect des le premier contact
**Afin de** savoir immediatement si j'interviens dans sa zone

## Fichiers concernes

- `apps/vitrine/src/components/ContactForm.tsx`

## Approche

Ajouter un champ "Ville ou code postal" dans le formulaire, entre le champ telephone et le champ service :

```tsx
<div>
  <label htmlFor="city" className="block text-sm font-medium text-gray-700 mb-2">
    Ville ou code postal
  </label>
  <input
    type="text"
    id="city"
    name="city"
    value={formData.city}
    onChange={handleChange}
    className="w-full px-4 py-3 border border-gray-300 rounded-lg ..."
    placeholder="Angers, 49000"
  />
</div>
```

- Champ optionnel (pas required)
- Placeholder "Angers, 49000"
- Ajouter `city` dans l'interface FormData et dans le body envoye a Web3Forms
- Reorganiser la grid : telephone | ville sur une ligne, service seul en dessous
