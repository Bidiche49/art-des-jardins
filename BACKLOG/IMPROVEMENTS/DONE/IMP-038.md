# IMP-038: Ajouter loading="lazy" systematique sur images non-hero

**Type:** Improvement
**Statut:** Fait
**Priorite:** Moyenne
**Complexite:** XS
**Tags:** vitrine, perf, seo
**Date creation:** 2026-02-13

---

## Description

Certaines images du site n'ont pas l'attribut `loading="lazy"`, ce qui fait que toutes les images sont chargees au chargement initial de la page, ralentissant le LCP (Largest Contentful Paint) et le score Core Web Vitals.

Seule l'image hero (preloadee dans layout.tsx) doit etre en `loading="eager"`. Toutes les autres doivent etre lazy.

## Fichiers concernes

- `apps/vitrine/src/app/services/page.tsx` - Images services (ont `loading="lazy"` OK)
- `apps/vitrine/src/components/ui/ServiceCard.tsx` - Verifier
- `apps/vitrine/src/components/ui/PhotoGallery.tsx` - Verifier
- `apps/vitrine/src/components/ui/InlineGallery.tsx` - Verifier
- `apps/vitrine/src/app/page.tsx` - Background images en CSS (ne supportent pas lazy)

## Approche

1. Auditer toutes les balises `<img>` dans apps/vitrine/src/
2. Ajouter `loading="lazy"` sur chaque `<img>` qui n'est pas dans le viewport initial
3. L'image hero doit rester `loading="eager"` ou sans attribut (preloadee)
4. Pour les background images CSS (`bg-[url(...)]`), envisager de les remplacer par des `<img>` avec lazy loading ou utiliser une technique d'intersection observer

## Criteres d'acceptation

- [ ] Toutes les `<img>` hors hero ont `loading="lazy"`
- [ ] L'image hero reste chargee immediatement
- [ ] Aucune image sous le fold n'est chargee au premier paint

## Tests de validation

- [ ] Lighthouse Performance score ameliore
- [ ] Network tab : verifier que les images ne sont chargees qu'au scroll
- [ ] LCP non impacte (image hero toujours rapide)
