# IMP-044: Remplacer background images CSS par des img avec lazy loading

**Type:** Improvement
**Statut:** Fait
**Priorite:** Moyenne
**Complexite:** M
**Tags:** vitrine, perf, seo
**Date creation:** 2026-02-13

---

## Description

Plusieurs sections CTA utilisent des background images en CSS inline (`bg-[url('/images/realisations/...')]`). Ces images :
- Ne supportent pas `loading="lazy"`
- Ne sont pas indexees par Google Images
- Ne beneficient pas de srcset/sizes pour le responsive
- Sont chargees meme si la section n'est jamais visible

Pages concernees :
- Homepage : section CTA finale
- paysagiste-angers : section CTA
- elagage-angers : section CTA
- services/[slug] : section CTA
- [serviceCity] : section CTA

- `apps/vitrine/src/app/page.tsx` (ligne 83)

## Approche

Remplacer le pattern :
```tsx
<div className="absolute inset-0 bg-[url('/images/realisations/creation-2-1200w.webp')] bg-cover bg-center" />
```

Par :
```tsx
<img
  src="/images/realisations/creation-2-1200w.webp"
  alt=""
  loading="lazy"
  className="absolute inset-0 w-full h-full object-cover"
  aria-hidden="true"
/>
```

Ou creer un composant reutilisable `BackgroundImage` :
```tsx
function BackgroundImage({ src, alt = '' }: { src: string; alt?: string }) {
  return (
    <img
      src={src}
      alt={alt}
      loading="lazy"
      className="absolute inset-0 w-full h-full object-cover"
      aria-hidden="true"
    />
  );
}
```

## Criteres d'acceptation

- [ ] Toutes les background images CSS remplacees par des `<img>`
- [ ] `loading="lazy"` sur chaque image de CTA
- [ ] L'overlay (gradient) fonctionne toujours au-dessus de l'image
- [ ] Le rendu visuel est identique
- [ ] Performance amelioree (images non chargees si hors viewport)
