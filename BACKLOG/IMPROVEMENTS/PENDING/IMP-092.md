# IMP-092: Couleur secondary (or/dore) plus presente dans le design

**Type:** Improvement
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** S
**Tags:** vitrine, design, premium
**Date creation:** 2026-02-27

---

## Description

Le site est trop "vert monochrome". La palette secondary (or/dore : secondary-400 `#c4ad72`, secondary-500 `#b89a58`) est definie dans tailwind.config.js mais sous-utilisee dans le design. Presque tous les accents sont en primary-600 vert ou en blanc/gris.

Pour un positionnement premium/haut de gamme, la couleur or devrait etre utilisee pour :
- Les accents de luxe (lignes decoratives, bordures)
- Les badges et etiquettes ("Devis gratuit", "Credit d'impot")
- Les etoiles de notation
- Les separateurs de section
- Les hover states secondaires
- Les icones de confiance

## User Story
**En tant que** visiteur
**Je veux** percevoir un site haut de gamme et chaleureux
**Afin de** associer l'entreprise a un service premium

## Criteres d'acceptation
- [ ] Au moins 3 elements utilisent la palette secondary
- [ ] Accent bar hero en secondary-500 (au lieu de primary-500)
- [ ] Badges "Devis gratuit" en secondary
- [ ] Coherence visuelle maintenue
- [ ] Pas de surcharge de couleurs

## Fichiers concernes
- `apps/vitrine/src/app/globals.css`
- Composants individuels (HeroSection, ServiceCard, trust band, etc.)
