# IMP-062: Loading states et skeletons

**Type:** Improvement
**Statut:** A faire
**Priorite:** Haute
**Complexite:** S
**Tags:** ux, performance
**Date creation:** 2026-02-25

---

## Description

Pages detail affichent juste AejSpinner() centre. AejShimmerCard et AejShimmerList existent mais ne sont pas utilisees. Overlay bloquant couvre tout l'ecran sur devis builder. Pas de timeout visuel (spinner infini).

## Findings associes

- STATE-001: Inconsistance loading (spinner vs shimmer)
- STATE-002: Pas de skeleton shimmer sur pages detail
- STATE-003: Loading overlay bloquante (modal opacity)
- STATE-004: Pas de timeouts visuels pour requetes longues
- STATE-009: Pas de distinction vide vs chargement sur search

## Criteres d'acceptation

- [ ] Shimmer skeletons sur toutes les pages detail (client, chantier, intervention, devis)
- [ ] Conserver ancien data pendant refresh (pas retour spinner)
- [ ] Remplacer overlay modal par floating indicator ou snackbar
- [ ] Message "Ca prend du temps..." apres 5s
- [ ] Option annuler apres 10s

- `apps/mobile/lib/shared/widgets/aej_shimmer.dart`
