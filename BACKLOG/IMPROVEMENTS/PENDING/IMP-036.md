# IMP-036: Ajouter images Open Graph specifiques aux pages manquantes

**Type:** Improvement
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** S
**Tags:** seo, vitrine, social
**Date creation:** 2026-02-13

---

## Description

Certaines pages n'ont pas d'image Open Graph specifique (contact, mentions legales, politique de confidentialite). Quand ces pages sont partagees sur les reseaux sociaux ou dans des messages, elles affichent l'image OG par defaut ou aucune image.

## Fichiers concernes

- `apps/vitrine/src/app/contact/page.tsx` - Pas d'image OG
- `apps/vitrine/src/app/mentions-legales/page.tsx` - Pas d'image OG
- `apps/vitrine/src/app/politique-confidentialite/page.tsx` - Pas d'image OG

## Approche

1. Utiliser l'image OG par defaut du site pour les pages legales
2. Creer une image OG specifique pour la page contact (si possible)
3. Ajouter dans chaque metadata :
```typescript
openGraph: {
  images: [{ url: '/images/og-image.jpg', width: 1200, height: 630 }],
}
```

Idealement, generer des images OG dynamiques avec `next/og` (ImageResponse), mais en mode `export` cela n'est pas possible. L'alternative est d'utiliser l'image par defaut.

## Criteres d'acceptation

- [ ] Toutes les pages ont une image OG
- [ ] L'image OG est correctement dimensionnee (1200x630)
- [ ] Le partage sur Facebook/LinkedIn affiche une image

## Tests de validation
