# IMP-068: Messages erreur user-friendly

**Type:** Improvement
**Statut:** A faire
**Priorite:** Haute
**Complexite:** S
**Tags:** ux, errors
**Date creation:** 2026-02-25

---

## Description

Erreurs affichees brutes ("Erreur: $error" ou "Erreur: $e"). AejErrorRetry a des factories (network, server, timeout) mais peu utilisees. Empty states sans call-to-action. Retry auto sans indication visuelle.

## Findings associes

- STATE-005: Messages d'erreur techniques au lieu d'user-friendly
- STATE-006: Pas de visual feedback pour retry automatique
- STATE-007: Pas de backoff visible
- STATE-008: Empty states generiques sans CTA

## Criteres d'acceptation

- [ ] Messages traduits: NetworkFailure → "Verifiez votre connexion internet"
- [ ] ServerFailure → "Le serveur ne repond pas. Reessayez dans quelques minutes."
- [ ] Utiliser AejErrorRetry.network/.server/.timeout partout (pas de raw Text)
- [ ] CTA dans tous les empty states ("+ Ajouter client", "+ Creer devis")
- [ ] Indication visuelle du retry auto ("Tentative 2/3...")
