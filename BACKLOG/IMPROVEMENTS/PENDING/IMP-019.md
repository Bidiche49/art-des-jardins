# IMP-019: Ameliorer accessibilite PWA (aria labels)

**Type:** Improvement
**Statut:** A faire
**Priorite:** Moyenne
**Complexite:** S
**Tags:** pwa, ux
**Date creation:** 2026-02-12

---

## Description

Seulement 22 occurrences d'attributs `aria-*` dans toute la PWA. Boutons sans label, inputs sans description pour screen readers, pas de `role` pour les composants custom. L'app est difficilement utilisable avec un lecteur d'ecran.

## Criteres d'acceptation

- [ ] `aria-label` sur tous les boutons icones (fermer, menu, actions)
- [ ] `aria-describedby` sur les inputs avec messages d'erreur
- [ ] `role="alert"` sur les messages d'erreur/succes
- [ ] `role="dialog"` sur les modales
- [ ] Modal.tsx : bouton fermer avec aria-label dynamique
- [ ] Score Lighthouse Accessibility > 90

## Fichiers concernes

- `apps/pwa/src/components/ui/Modal.tsx:78-95` - bouton fermer
- `apps/pwa/src/components/ui/Button.tsx` - boutons icones
- Tous les formulaires (Login, DevisBuilder, etc.)

## Analyse / Approche

```typescript
// Modal close button
<button onClick={onClose} aria-label="Fermer la modal">

// Input avec erreur
<input aria-describedby={error ? `${name}-error` : undefined} />
{error && <p id={`${name}-error`} role="alert">{error}</p>}
```

## Tests de validation

- [ ] Audit Lighthouse Accessibility > 90
- [ ] Navigation au clavier possible dans toute l'app
- [ ] Lecteur d'ecran annonce correctement les boutons et erreurs
