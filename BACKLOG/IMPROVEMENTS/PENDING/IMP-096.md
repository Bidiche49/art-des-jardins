# IMP-096: dns-prefetch et preconnect pour ressources tierces

**Type:** Improvement
**Statut:** A faire
**Priorite:** Basse
**Complexite:** XS
**Tags:** vitrine, perf
**Date creation:** 2026-02-27

---

## Description

Le site charge des ressources depuis des origines tierces (Google Maps iframe, WhatsApp, Instagram, analytics potentiel) mais aucun `dns-prefetch` ni `preconnect` n'est configure pour accelerer la resolution DNS et l'etablissement de connexion.

### Resource hints a ajouter
```html
<link rel="dns-prefetch" href="//maps.google.com" />
<link rel="dns-prefetch" href="//maps.gstatic.com" />
<link rel="dns-prefetch" href="//wa.me" />
<link rel="preconnect" href="https://maps.google.com" crossorigin />
```

Si analytics configure (Plausible/Umami), ajouter aussi le domaine d'analytics.

## Criteres d'acceptation
- [ ] dns-prefetch pour toutes les origines tierces utilisees
- [ ] preconnect pour les origines les plus critiques
- [ ] Ajoute dans le layout.tsx ou via metadata
- [ ] Pas de hints vers des origines non utilisees
